# 設計メモ: 成績表示のUX配慮

## 背景

セットマージャンの成績を正確にデータ化すること自体にはリスクがある。
負けが込んでいるメンバーにとって、通算成績がグラフや数字で明確に可視化されると
心理的な負担になり、最悪の場合メンツが集まらなくなる。

セットマージャンの本質は仲間との娯楽であり、勝敗記録はあくまでスパイス。
アプリの設計はこの前提を尊重する必要がある。

## 設計方針: 「見たい人だけが見に行ける」構造

### 対局動線と分析動線を分離する

**対局中に自然と目に入る情報（メイン動線）:**
- 今日のセッションへのアクセス
- 進行中の帳表（その日のスコアのみ）
- 清算結果

**自分から見に行かないと見えない情報（サブ動線）:**
- 通算収支・累計ポイント
- 平均順位・順位分布グラフ
- レーティング・ランキング
- 対戦相性マトリクス
- 月別成績推移

### 具体的なUI案

- ホーム画面には進行中セッションと直近の対局日だけを表示し、通算成績サマリーは置かない
- 下部タブは「ホーム・対局・履歴・設定」の4つに絞る
- 「分析」「レーティング」は履歴または設定画面の中に階層を下げて配置する
- 個人成績ダッシュボードへの導線はメンバー詳細画面からのみ（一覧画面には通算成績を表示しない）

### データは残す、見せ方を制御する

データの蓄積自体は全て行う（後から分析できる価値は大きいため）。
ただし、対局のために集まった場面で不要にネガティブな情報が目に入らないよう、
表示の階層設計で配慮する。

## 将来的な検討事項

- 成績表示のトーンを工夫する（累計収支よりも「直近5回の平均順位」「前回からの改善」など、成長にフォーカスした指標を強調）
- レーティング表示をオプション化（グループ設定でON/OFF）
- 分析画面へのアクセスにワンクッション入れる（「通算成績を表示しますか？」の確認）
