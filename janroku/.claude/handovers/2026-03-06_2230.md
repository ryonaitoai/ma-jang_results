## 重要な決定事項
- **better-sqlite3 → @libsql/client に変更**: Visual Studio Build Tools がない環境のため。ネイティブビルド不要
- **ポイント入力モードがデフォルト**: 実運用フロー（2〜4着が口頭申告、トップは逆算）に最適化
- **DBスキーマ**: `hanchan_scores.raw_score` は nullable（ポイント入力モードでは null）
- **論理削除**: 半荘は `is_void` フラグ、メンバーは `is_active` フラグで管理
- **セッションにルールをインラインスナップショット**: プリセット変更が過去データに影響しない設計

## 発見した問題・注意点
- **Node.js v24 + Windows**: better-sqlite3 のプリビルドバイナリが存在せず、node-gyp ビルドも失敗する → libsql で回避
- **Drizzle マイグレーション**: カラム追加を含むスキーマ変更時、既存データからの SELECT に新カラムを含めてしまう → マイグレーションSQLの手動修正が必要だった
- **TypeScript strict mode + Map イテレーション**: `target: es2017` を tsconfig に追加して解決
- **Bash コマンドの権限**: ユーザーの権限設定により一部コマンド（rm -f, npx create-next-app）が拒否される場合あり
